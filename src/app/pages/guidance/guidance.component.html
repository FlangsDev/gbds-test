Clean Code Principle

SOLID, KISS, DRY — prinsip dasar penulisan kode yang bersih dan maintainable.

SOLID Principle

  • S — Single Responsibility: Class hanya memiliki satu tanggung jawab.
  • O — Open/Closed: Terbuka untuk ekstensi, tertutup untuk modifikasi.
  • L — Liskov Substitution: Subtype harus bisa menggantikan base type.
  • I — Interface Segregation: Interface kecil dan fokus.
  • D — Dependency Inversion: Modul high-level tidak bergantung pada low-level.

KISS Principle

"Keep It Simple, Stupid" — Hindari kompleksitas yang tidak perlu. Rancang solusi yang sederhana, fungsional, dan mudah dipahami.

DRY Principle

"Don't Repeat Yourself" — Hindari duplikasi kode. Gunakan abstraksi dan reuse code untuk mengurangi redundansi.

export abstract class BaseComponent implements OnDestroy {
  unsubscribe$: Subject<void> = new Subject<void>();
  ngOnDestroy(): void { this.unsubscribe$.next(); }
}

Git Versioning & Rules

Aturan commit message, branching strategy (Git-flow), tagging, dan pull request.

Commit Message

  • Gunakan bracket: [add], [update], [remove]
  • Gunakan present tense & imperative mood
  • Subject line max 50 karakter
  • Capitalize huruf pertama
[add] color change logic on button
[update] layout design
[remove] deprecated function

Git-flow Branches

  • master — Production-ready code
  • staging — Staging changes
  • development — Main development
  • feature/* — New features

Pull Request

  • Buka PR setelah push ke branch
  • Diskusi dan review dengan collaborator
  • Tambahkan follow-up commits jika perlu
  • Merge ke base branch setelah approval

Best Practices

  • Gunakan single repository
  • Gunakan tags untuk releases
  • Hindari binary files di repo
  • Jangan simpan password di repository

COPY target/*.jar app.jar
